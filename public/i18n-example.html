WSH i18n · Integration reference

Sistema de
traducción

Demostración del sistema i18n para World Stone Hub. Cambia el idioma con el selector de la barra de navegación y observa cómo todos los textos marcados se actualizan al instante, sin recargar la página.

1 · Texto simple — data-i18n

Añade data-i18n="clave.ruta" a cualquier elemento. El motor reemplaza su textContent con la traducción.

2 · Atributos — data-i18n-attr

Para traducir placeholder, title, aria-label u otros atributos, usa data-i18n-attr="atributo:clave[,atributo2:clave2]".

Live demo
← Hover / inspect este enlace para ver title y aria-label
HTML source
<input
  placeholder="Buscar..."
  data-i18n-attr="placeholder:search.placeholder"
>

<!-- Multiple attrs, comma-separated -->
<a
  data-i18n-attr="title:nav.mapaAriaLabel,
             aria-label:nav.mapaAriaLabel"
>...</a>

3 · API JavaScript

Usa WSH_i18n.t(key) para obtener traducciones en código JS (útil en renderizadores Supabase, mensajes de error, etc.). 

// Get translated string
const label = WSH_i18n.t('stats.empresas');
// ES → "Empresas"   EN → "Companies"

// With fallback if key missing
const txt = WSH_i18n.t('state.cargando', '...');

// Current language
const lang = WSH_i18n.getLang(); // 'es' or 'en'

// Switch language (returns Promise)
WSH_i18n.setLang('en').then(function() {
  console.log('Language switched');
});

// Translate a freshly-rendered Supabase card
function renderCard(data) {
  const el = document.createElement('div');
  el.innerHTML = `
    <span data-i18n="card.verFicha">Ver ficha</span>
    <span data-i18n="card.verificado">Verificado</span>
  `;
  WSH_i18n.applyTo(el);   // ← translates only this fragment
  document.body.appendChild(el);
}

// React to language changes (e.g. re-render dynamic content)
document.addEventListener('wsh:langchange', function(e) {
  console.log('New lang:', e.detail.lang);
  // re-fetch or re-render Supabase content if needed
});

4 · Contenido dinámico (Supabase)

Cuando el contenido de las tarjetas se genera con JS desde Supabase, llama a WSH_i18n.applyTo(el) sobre el fragmento recién insertado para que se traduzcan los atributos estáticos de la UI. 

/* In your Supabase render function */
function renderProveedores(rows) {
  const grid = document.getElementById('cards-grid');
  grid.innerHTML = rows.map(function(p) {
    return `
    <div class="card">
      <div class="card-nombre">${p.nombre_empresa}</div>
      <span class="card-tipo">${p.tipo_empresa}</span>

      <!-- Static UI labels → translate with data-i18n -->
      <span class="label" data-i18n="card.fundada">Fundada</span>
      <span>${p.ano_fundacion}</span>

      <a href="proveedor.html?id=${p.slug}"
         data-i18n="card.verFicha">Ver ficha</a>
    </div>
    `;
  }).join('');

  /* Apply i18n to the freshly rendered HTML */
  WSH_i18n.applyTo(grid);
}

5 · Añadir a una página existente

Tres pasos para integrar i18n en cualquier página HTML de WSH. 

<!-- PASO 1: Cargar el script antes de </body>
     (después de tu JS de Supabase) -->
<script src="/i18n.js"></script>

<!-- Si tu repo está en un subdirectorio (GitHub Pages sin dominio propio),
     configura el basePath antes de cargar el script: -->
<script>
  window.WSH_i18n_config = { basePath: '/worldstonehub/' };
</script>
<script src="/worldstonehub/i18n.js"></script>

<!-- PASO 2: Añadir el selector al nav, después de </ul> -->
<div class="lang-switcher"
     role="group"
     data-i18n-attr="aria-label:lang.cambiar">
  <button class="lang-btn active" data-lang="es"
          aria-pressed="true"
          data-i18n-attr="aria-label:lang.es">ES</button>
  <span class="lang-sep" aria-hidden="true"></span>
  <button class="lang-btn" data-lang="en"
          aria-pressed="false"
          data-i18n-attr="aria-label:lang.en">EN</button>
</div>
